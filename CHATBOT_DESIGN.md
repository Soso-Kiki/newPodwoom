# 챗봇 설계 문서

## 두 챗봇의 역할 분리

### 1. 캐릭터와 대화
**목적**: 감성적 관계, 장기 기억, 페르소나 → 유저 리텐션

```
Q → 페르소나(투자성향/캐릭터) → DB(과거매매이력 + 장기기억 3일) → A → 저장
```

- tool_use로 매매이력/일지/투자성향 조회
- 장기기억 3일 이내 대화 참조
- 응답을 SQLite + ChromaDB에 저장
- 서버: Node.js Express (port 3001) + Claude API

### 2. AI 챗봇에게 물어보기
**목적**: 즉각적인 종목 분석 → 매매 의사결정 지원

```
Q → 페르소나(X) → 현재 데이터 + DB(과거 매매이력 + 매매일지) → A → 저장X
```

컨텍스트로 주입되는 데이터:
- 현재가, 평균매입가, 수익률, 보유기간
- 매매포인트 AI요약(aiLine), 승률, 52주위치
- 유사도 분석 결과 (score, 매도 후 7일)
- 과거 매매이력 전체 (해당 종목)
- 매매일지 aiData (buy_pattern, sell_pattern, hold_analysis)

- 서버: FastAPI (port 8000) + Gemini API

## 개선 아이디어 (미구현)

### 티키타카 - 유저 인터뷰형 대화

#### 매도 후 회고 인터뷰 (우선순위 1위)
매도 기록 생성 시 캐릭터가 짧은 인터뷰 시작:
```
캐릭터: "카카오 +8% 익절했네요. 왜 여기서 팔았어요?"
유저 답변 → AI 회고 요약 → 매매일지 "심리 메모"로 저장
```
현재 매매일지에 심리/감정 레이어 추가 가능.

#### 매수 전 체크인
종목 페이지 진입 시 캐릭터가 먼저 질문:
```
캐릭터: "삼성전자 보고 있네요. 
         저번에 여기서 -12% 물린 적 있잖아요.
         이번엔 뭘 다르게 할 건가요?"
```
유저가 매수 근거를 언어화하게 유도 → 충동매매 방지.

#### 손절 망설임 감지
보유 종목 -5% 이하 시:
```
캐릭터: "원칙에 -5% 손절이라고 적혀 있는데, 지금 어떤 생각이에요?"
```
대화로 손절 라인 재설정 유도.

## 장기기억 설계

### 현재 구조
- 단기기억: 최근 3일 대화 (SQLite + ChromaDB)
- 검색: SQLite 키워드 우선 + ChromaDB 시맨틱 보완

### 개선 방향 (미구현)
- **단기기억**: 최근 3일 대화 (현재 구조 유지)
- **영구기억**: 투자성향/원칙/중요 결정 (날짜 무관 보존)
  - 예: "손절 못하는 성향", "카카오를 선호함"
  - 유저가 특정 키워드 언급 시 별도 태그로 저장

## LLM 의존도 낮추기 방향

### 즉시 적용 가능
1. **분석 결과 1회 생성 후 DB 고정** - 매번 LLM 호출 → 최초 1회만
2. **RAG → SQL 대체** - 구조화 데이터는 ChromaDB 대신 SQLite 직접 조회

### 다음 단계
3. **유저 직접 태그** - 매도 이유를 LLM 추론 대신 유저가 직접 선택
   ```
   [목표가 달성] [공포] [지루함] [손절] [급전필요]
   ```
4. **규칙 기반 텍스트** - aiLine, 패턴 요약을 조건 매칭 템플릿으로 대체

### LLM이 반드시 필요한 영역
- 캐릭터와 대화 (자연스러운 페르소나 대화)
- 복잡한 분석 질문 처리
